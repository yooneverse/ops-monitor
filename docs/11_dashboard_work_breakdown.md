# 대시보드 작업 분해

## 목표

대시보드가 단순 카드 모음이나 연습용 화면처럼 보이지 않도록 하고, 한 번에 전체를 갈아엎지 않아도 영역별로 개선할 수 있도록 작업 단위를 나눕니다.

이 문서는 `app/api/dashboard_page.py`의 현재 구조를 기준으로 대시보드 개선 작업을 쪼개는 용도로 작성했습니다.

## 현재 구성 섹션

### 1. 헤더

대상 함수:

- `build_page_header()`
- `build_utility_bar()`
- `build_sidebar()`

주요 포인트:

- 대시보드 첫인상
- 관리자 콘솔 느낌
- 사이드바 메뉴 체계
- 상단 유틸리티 정보 밀도

### 2. 상단 상태 구역

대상 함수:

- `build_summary_surface()`

주요 포인트:

- API, DB, 모니터링, 디스코드, 설정 상태를 한 블록으로 묶어 읽기 쉽게 유지
- 메모리/디스크는 퍼센트 성격이 강하므로 도넛 차트 영역으로 분리
- 핵심 장애 징후와 참고 지표를 섞지 않기

### 3. 세부 운영 패널

대상 함수:

- `build_detail_surface()`

주요 포인트:

- 최근 알림 영역 안에서 로그 갱신 버튼을 해결
- 경고, 로그, 보조 링크의 위계를 분명히 분리
- "설명하는 화면"보다 "확인하고 바로 행동하는 화면"에 가깝게 유지

### 4. 공통 스타일

대상 함수:

- `build_dashboard_styles()`

주요 포인트:

- 흰색과 푸른 계열을 기본으로 하되 과한 장식은 줄이기
- 패널 여백과 줄바꿈을 정제해서 업무 화면처럼 보이게 하기
- 사이드바/상단바/본문의 분리감 유지
- 관리자 콘솔다운 밀도와 톤 유지
- 반응형 동작

### 5. 인터랙션/문구

대상 함수:

- `build_dashboard_script()`

주요 포인트:

- 상태 용어를 전부 한글로 통일
- 로딩/빈 상태/오류 상태 문구를 짧게 유지
- 사이드바 검색과 필터링이 실제로 동작하도록 유지
- 더미 숫자나 움직이지 않는 값은 넣지 않기
- 연결 실패 시에도 화면이 완전히 비지 않도록 실패 표현을 분리

## 추천 작업 순서

1. 상단 상태 구역 재정리
2. 최근 알림 패널과 로그 갱신 동작 보강
3. 사이드바와 검색 인터랙션 연결
4. 공통 스타일과 줄바꿈 정제
5. 불필요한 설명 문구와 더미 값 제거

## 작업 원칙

대시보드 수정은 가능하면 한 번에 여러 영역을 섞지 말고, 위 섹션 중 하나를 기준으로 커밋 단위를 끊는 것이 좋습니다.

예시:

- 상단 상태 블록만 재배치
- 최근 알림 패널과 갱신 버튼만 수정
- 사이드바 검색과 필터만 연결

이렇게 나누면 디자인 변경이 커져도 추적과 되돌리기가 쉬워지고, "바이브 코딩으로 한 번에 만진 화면"이 아니라 의도를 가지고 정리한 흔적이 남습니다.
