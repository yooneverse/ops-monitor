# 대시보드 작업 분해

## 목표

대시보드가 기본 관리자 페이지처럼 보이는 상태를 벗어나고, 한 번에 전체를 갈아엎지 않아도 영역별로 개선할 수 있도록 작업 단위를 나눕니다.

이 문서는 `app/api/dashboard_page.py`의 현재 구조를 기준으로 대시보드 개선 작업을 쪼개는 용도로 작성했습니다.

## 현재 구성 섹션

### 1. 헤더

대상 함수:

- `build_page_header()`
- `build_utility_bar()`
- `build_sidebar()`

주요 포인트:

- 대시보드 첫인상
- 관리자 콘솔 느낌
- 사이드바 메뉴 체계
- 상단 유틸리티 정보 밀도

### 2. 운영 요약 패널

대상 함수:

- `build_summary_surface()`

주요 포인트:

- 상단 KPI 배치
- 어떤 지표를 1순위로 보여줄지
- 핵심 상태와 보조 상태의 시각적 차이

### 3. 세부 현황 패널

대상 함수:

- `build_detail_surface()`

주요 포인트:

- 경고 영역과 최근 알림 영역의 균형
- 버튼 위치와 행동 흐름
- 관리 화면다운 정보 위계

### 4. 공통 스타일

대상 함수:

- `build_dashboard_styles()`

주요 포인트:

- 색 체계
- 패널 여백
- 사이드바/상단바/본문의 분리감
- 관리자 콘솔다운 밀도와 톤
- 반응형 동작

### 5. 인터랙션/문구

대상 함수:

- `build_dashboard_script()`

주요 포인트:

- 상태 용어
- 로딩/빈 상태/오류 상태 문구
- 자동 갱신 안내
- 알림 메타데이터 표현 방식

## 추천 작업 순서

1. 운영 요약 패널 재정리
2. 최근 알림 패널 시각 계층 보강
3. 사이드바와 상단바의 관리자 콘솔 톤 보강
4. 공통 스타일 시스템 정제
5. 경고 영역 동작과 문구 다듬기

## 작업 원칙

대시보드 수정은 가능하면 한 번에 여러 영역을 섞지 말고, 위 섹션 중 하나를 기준으로 커밋 단위를 끊는 것이 좋습니다.

예시:

- 요약 패널만 재배치
- 최근 알림 패널만 스타일 보강
- 사이드바 정보 구조만 수정

이렇게 나누면 디자인 변경이 커져도 추적과 되돌리기가 쉬워집니다.
